Abstract

MusicXML is a standard open format for exchanging digital sheet music. It is designed for sharing sheet music files between applications, and for archiving sheet music files for use in the future. As of this publication date it is supported by over 200 applications.

Version 3.1 is the first version released by the W3C Music Notation Community Group. Previous MusicXML versions were released by the Recordare and MakeMusic companies.

                1. MusicXML Schema Files

                MusicXML 3.1 does not have a specification document separate from its schema files. The schema files for MusicXML 3.1 are available on the GitHub repository at https://github.com/w3c/musicxml/releases/tag/v3.1.

                2. Changes From MusicXML 3.0

                2.1 Licensing Changes

                MusicXML 3.1 is now licensed under the W3C Community Final Specification Agreement rather than the MusicXML Public License 3.0.

                2.2 Packaging Changes

                MusicXML 3.1 now recommends .musicxml rather than .xml as the file extension for uncompressed MusicXML files.

                MusicXML 3.1 adds a new file called mimetype as the first file in the zip container of a compressed MusicXML file. The file is uncompressed and has the string application/vnd.recordare.musicxml as its content.

                2.3 Schema Additions

                MusicXML 3.1 adds new features to MusicXML 3.0 score-partwise and score-timewise documents. These changes are included in the musicxml.xsd and to30.xsl files, as well as the following DTD module files.

                2.3.1 Changes in attributes.mod

                2.3.1.1 New Elements
                • The except-voice element is used to specify a combination of slash notation and regular notation.
                2.3.1.2 New Attributes
                • The id attribute has been added to the clef, key, measure-style, time, and transpose elements.
                • The smufl attribute has been added to the key-accidental element.
                2.3.1.3 New Values
                • New values double-sharp-down, double-sharp-up, flat-flat-down, flat-flat-up, arrow-down, arrow-up, and other have been added to the accidental-value type, used by the key-accidental element.

                2.3.2 Changes in barline.mod

                2.3.2.1 New Attributes
                • The id attribute has been added to the barline element.

                2.3.3 Changes in common.mod

                2.3.3.1 New Elements
                • The n element represents an n dynamic.
                • The pf element represents a pf dynamic.
                • The sfzp element represents an sfzp dynamic.
                2.3.3.2 New Attributes
                • The id attribute has been added to the coda, dynamics, fermata, and segno elements.
                • The smufl attribute has been added to the accidental-text, coda, other-dynamics, and segno elements.
                2.3.3.3 New Values
                • New values double-sharp-down, double-sharp-up, flat-flat-down, flat-flat-up, arrow-down, arrow-up, and other have been added to the accidental-value type, used by the accidental-text element.
                • New values pentagon, hexagon, heptagon, octagon, nonagon, and decagon have been added to the enclosure-shape type, used by the enclosure attribute.
                • New values double-angled, double-square, double-dot, half-curve, and curlew have been added to the fermata-shape type, used by the fermata element.

                2.3.4 Changes in direction.mod

                2.3.4.1 New Elements
                • The beat-unit-tied element represents tied notes in regular metronome marks.
                • The metronome-arrows element indicates the presence of metric modulation arrows on both sides of the metronome mark.
                • The metronome-tied element represents tied-notes in metric modulations and other metric relationships.
                • The staff-divide element represents staff division arrow symbols.
                • The symbol element specifies a musical symbol using a canonical SMuFL glyph name. It is used when an occasional musical symbol is interspersed into text.
                2.3.4.2 New Attributes
                • The id attribute has been added to the direction, direction-type, rehearsal, words, wedge, dashes, bracket, pedal, metronome, octave-shift, harp-pedals, damp, damp-all, eyeglasses, string-mute, scordatura, image, principal-voice, accordion-registration, percussion, other-direction , harmony, frame, grouping, print, and sound elements.
                • The smufl attribute has been added to the glass, pitched, other-direction, and other-percussion elements.
                • The height and width attributes have been added to the image element.
                • The abbreviated and number attributes have been added to the pedal element.
                • The parentheses and dashed-circle attributes have been added to the stick element.
                2.3.4.3 New Values
                • The sostenuto value has been added to the pedal-type type, used by the type attribute of the pedal element.
                • New values drum stick, slide brush on gong, and superball have been added to the beater-value type, used by the beater element.
                • New values lotus flute and megaphone have been added to the effect type, used by the effect element.
                • New values glass harmonica and glass harp have been added to the glass-value type, used by the glass element.
                • New values Chinese tomtom, cuica, Indo-American tomtom, Japanese tomtom, and tabla have been added to the membrane type, used by the membrane element.
                • New values agogo, bell tree, cencerro, chain rattle, jaw harp, jingle bells, musical saw, shell bells, and tam tam with beater have been added to the metal type, used by the metal element.
                • New values celesta, lithophone, steel drums, and tubaphone have been added to the pitched-value type, used by the pitched element.
                • New values glockenspiel, gum, hammer, superball, and wound have been added to the stick-type type, used by the stick-type element.
                • New values bamboo scraper, castanets with handle, football rattle, quijada, rainstick, reco-reco, and whip have been added to the wood type, used by the wood element.

                2.3.5 Changes in layout.mod

                2.3.5.1 New Elements
                • The glyph element represents what SMuFL glyph should be used for different variations of symbols that are semantically identical.

                2.3.6 Changes in note.mod

                2.3.6.1 New Elements
                • A single note element may now have both cue and grace child elements.
                • The arrowhead element represents arrowheads without an arrow stem.
                • The brass-bend element represents the u-shaped bend symbol used in brass notation.
                • The flip element represents the flip symbol used in brass notation.
                • The golpe element represents the golpe symbol that is used for tapping the pick guard in guitar music.
                • The half-muted element represents the half-muted symbol which looks like a circle with a plus sign inside.
                • The harmon-mute element represents the symbols used for harmon mutes in brass notation.
                • The haydn element represents the Haydn ornament, defined in SMuFL as ornamentHaydn.
                • The inverted-vertical-turn element represents the turn symbol shape arranged vertically going from upper left to lower right.
                • The open element represents the open symbol, which looks like a circle.
                • The smear element represents the tilde-shaped smear symbol used in brass notation.
                • The soft-accent element indicates a soft accent that is not as heavy as a normal accent. It is often notated as <>.
                • The figure element may now have the footnote and level elements as optional child elements.
                2.3.6.2 New Attributes
                • The id attribute has been added to the note, beam, notations, tied, slur, tuplet, glissando, slide, other-notation, ornaments, accidental-mark, technical, articulations, arpeggiate, non-arpeggiate, lyric, and figured-bass elements.
                • The bracket, parentheses, and size attributes have been added to the accidental-mark element.
                • The time-only attribute has been added to the lyric element.
                • The print-leger attribute has been added to the note element.
                • The hand attribute has been added to the tap element.
                • The line-length attribute has been added to the doit, falloff, plop, and scoop elements.
                • The smufl attribute has been added to the accidental, accidental-mark, arrow, elision, notehead, stopped, tremolo, other-articulation, other-notation, other-ornament, and other-technical elements.
                2.3.6.3 New Values
                • New values double-sharp-down, double-sharp-up, flat-flat-down, flat-flat-up, arrow-down, arrow-up, and other have been added to the accidental-value type, used by the accidental and accidental-mark elements.
                • The let-ring value has been added to the type attribute of the tied element.
                • The unmeasured value has been added to the type attribute of the tremolo element.
                • The circled and other values have been added to notehead-value type, used by the notehead element.
                • The belltree value has been added to the handbell-value type, used by the handbell element.
                • The salzedo and upbow values have been added to the breath-mark-value type, used by the breath-mark element.
                • The caesura element uses the new caesura-vale type with values of normal, thick, short, curved, and single. An empty string value is included for compatibility with MusicXML 3.0.
                • The grace-cue value has been added to the symbol-size type, used by the size attribute.

                2.3.7 Changes in score.mod

                2.3.7.1 New Elements
                • The credit-symbol element specifies a musical symbol using a canonical SMuFL glyph name. It is used when an occasional musical symbol is interspersed into text.
                2.3.7.2 New Attributes
                • The id attribute has been added to the credit, credit-image, credit-words, and measure elements.
                • The height and width attributes have been added to the credit-image element.
                • The text attribute has been added to the measure element.

                2.4 Removed and Deprecated Features

                • The font-family, font-size, font-style, and font-weight attributes have been removed from the extend element in the note.mod and musicxml.xsd files. The extend element contains no text, and these attributes were added by mistake in MusicXML 1.1.
                • The bezier-offset and bezier-offset2 attributes in the common.mod and musicxml.xsd files are deprecated as of MusicXML 3.1.

                2.5 Documentation Changes

                MusicXML 3.1 made the following changes to the XSD and DTD schema documentation.

                • The DTD documentation for the actual-notes and normal-notes elements has been corrected to fix a typo.
                • The XSD and DTD documentation for the accordion-high, accordion-low, and accordion-middle elements clarifies that the elements are omitted if no dots are present.
                • The XSD and DTD documentation for the attack and release attributes and their interrelationship has been clarified.
                • The XSD and DTD documentation for the bezier-x, bezier-y, bezier-x2, bezier-y2, bezier-offset, and bezier-offset2 attributes and their interrelationships has been clarified.
                • The cancel complex type XSD documentation now describes the location attribute correctly.
                • The cue element XSD and DTD documentation clarifies that the cue element indicates silent notes.
                • The direction element XSD and DTD documentation clarifies the relationship between direction and note elements.
                • The falloff element XSD documentation has been corrected to indicate the falloff comes after the note.
                • The interchangeable complex type XSD documentation no longer refers to a non-existent parentheses attribute.
                • The key-octave element XSD and DTD documentation clarifies the behavior of the cancel attribute.
                • The multiple-rest complex type XSD documentation no longer refers to a non-existent type attribute.
                • The note-type complex type XSD documentation now refers to 1024th to maxima instead of 256th to long.
                • The number-level simple type XSD documentation now refers to an optional rather than implied value.
                • The string-number simple type and string element XSD and DTD documentation clarifies how string numbers are ordered.
                • The tied element XSD and DTD documentation clarifies how to represent different types of single-ended tie symbols.
                • The wavy-line XSD and DTD documentation now refers to barline rather than measure elements.

                The history of all the changes from MusicXML 3.0 to MusicXML 3.1 can be found in the GitHub issue list for the v3.1 milestone.
